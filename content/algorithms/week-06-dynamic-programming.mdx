# Week 6 — Dynamic Programming

## Learning Objectives

- Understand the basic idea of dynamic programming.
- Compare naive recursion with dynamic programming on Fibonacci numbers.
- Apply dynamic programming to assembly line scheduling.
- Apply dynamic programming to the 0-1 knapsack problem.

## Core Concepts

### Why dynamic programming?

The lecture motivates dynamic programming by showing repeated computation in naive recursive Fibonacci evaluation. For example, computing `F(7)` repeats smaller subproblems many times.

### Basic methodology

The lecture summarizes the method as:

1. Write a formula relating the problem to smaller subproblems.
2. Index the subproblems so they can be stored in a table.
3. Fill the table bottom-up.

### Fibonacci example

The recursive definition shown is:

- `F(n) = 1` if `n = 0` or `n = 1`
- `F(n) = F(n-1) + F(n-2)` if `n > 1`

The recursive version is exponential because it repeats work. The dynamic programming version computes values once in order from small to large.

### Assembly line scheduling

The lecture defines:

- `f1[j]`: fastest time to get through station `S1,j`
- `f2[j]`: fastest time to get through station `S2,j`

The recurrence compares staying on the same line with transferring from the other line.

### 0-1 knapsack

The lecture defines `f[i, w]` as the maximum profit obtainable from items `{1, ..., i}` with weight limit `w`.

The recurrence considers two cases:

- do not take item `i`
- take item `i`, if allowed by the weight bound

## Key Algorithms

### Bottom-up Fibonacci

Store previously computed Fibonacci values in an array and build upward.

### Assembly line dynamic programming

Compute `f1[j]` and `f2[j]` incrementally from station `1` to station `n`.

### 0-1 knapsack dynamic programming

Fill a two-dimensional table using the recurrence for `f[i, w]`.

## Pseudocode

```text
Procedure F(n)
  Set A[0] = A[1] = 1
  for i = 2 to n do
    A[i] = A[i-1] + A[i-2]
  return A[n]
```

```text
set f1[1] = a1,1
set f2[1] = a2,1
for j = 2 to n do
begin
  set f1[j] = min(f1[j-1] + a1,j, f2[j-1] + t2,j-1 + a1,j)
  set f2[j] = min(f2[j-1] + a2,j, f1[j-1] + t1,j-1 + a2,j)
end
set f* = min(f1[n], f2[n])
```

```text
f[i, w] =
  0                              if i = 0 or w = 0
  f[i-1, w]                      if wi > w
  max(f[i-1, w], vi + f[i-1, w-wi]) otherwise
```

## Time Complexity

- Recursive Fibonacci: exponential time according to the lecture.
- Dynamic programming Fibonacci: `O(n)`.
- Assembly line scheduling: `O(n)`.
- 0-1 knapsack dynamic programming: `O(nW)`, which the lecture calls pseudo-polynomial.

## Worked Example

For assembly line scheduling, the lecture works through a four-station example and computes:

- `f1[1] = 5`, `f2[1] = 15`
- `f1[2] = 10`, `f2[2] = 11`
- `f1[3] = 19`, `f2[3] = 14`
- `f1[4] = 20`, `f2[4] = 21`

So the optimal total time is:

`f* = min(f1[4], f2[4]) = 20`

For the knapsack example with capacity `15`, the lecture's final optimal item set is `{2, 3, 4, 5}` with maximum value `15`.

## Common Mistakes

- Using naive recursion when many subproblems repeat.
- Writing a recurrence without defining the subproblem state clearly.
- Filling the DP table in an order that does not respect dependencies.
- Mistaking `O(nW)` for polynomial in the size of the input.
